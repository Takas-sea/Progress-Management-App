# UI プロトタイプ確認ガイド

**目的**: リマインダー通知設定 & マイルストーン達成表示 の UI/UX を実装前に確認し、フィードバックを得る

---

## 🚀 プロトタイプの起動

### 1. アプリケーション開始

```bash
npm run dev
```

### 2. プロトタイプページを開く

ブラウザで以下の URL にアクセス：

```
http://localhost:3000/prototype
```

---

## 📋 確認項目

### **リマインダー通知設定** 🔔

このタブで以下を確認してください：

#### UI/UX
- [ ] **ボタン配置**: 「保存」「テスト送信」ボタンの位置は使いやすいか？
- [ ] **色分け**: 有効/無効の状態が視覚的に分かりやすいか？
- [ ] **トグル**: 通知開閉のトグルスイッチは明確か？
- [ ] **時間入力**: time input の表示と使いやすさは？
- [ ] **曜日選択**: 横 7 個のボタンでの曜日選択は良いか？下にテキスト表示は必要か？

#### 機能性
- [ ] **設定項目の多さ**: 選択肢が多すぎたり、少なかったりしないか？
- [ ] **デフォルト値**: デフォルト値（19:00、月-金）は適切か？
- [ ] **保存確認**: "設定を保存しました" の確認表示は分かりやすいか？
- [ ] **テスト通知**: テスト送信の説明文は十分か？

#### 改善案
- [ ] **追加したい項目**: 何か足りない機能はあるか？
- [ ] **削除したい項目**: 不要に思える項目はあるか？
- [ ] **順序変更**: 設定項目の順番は最適か？
- [ ] **デザイン**: フォントサイズ・間隔・色は最適か？

---

### **マイルストーン達成表示** 🏆

このタブで以下を確認してください：

#### UI/UX
- [ ] **セクション分け**: 達成済み / 進行中 の分け方は分かりやすいか？
- [ ] **アイコン**: ⭐ / ⏳ のアイコンは分かりやすいか？
- [ ] **プログレスバー**: 進捗状況の表示は見やすいか？
- [ ] **色選択**: 緑（達成）/ 青（進行中）の色分けは良いか？

#### 情報表示
- [ ] **達成日**: 達成日時の表示形式は分かりやすいか？
- [ ] **進捗表示**: "7/14" や "142.5/200" の形式は良いか？
- [ ] **パーセンテージ**: 進捗率の表示は必要か？
- [ ] **統計情報**: 「現在のストリーク」「累計学習時間」の表示は有用か？

#### マイルストーンの粒度
- [ ] **ストリーク**: 7, 14, 30, 100 日のマイルストーンは適切か？
  - 増やして欲しい（例: 21, 60 日）？
  - 減らして欲しい？
- [ ] **学習時間**: 100, 200, 300, 500 時間は適切か？
  - 増やしたい？減らしたい？

#### 改善案
- [ ] **追加表示**: 他に表示したい情報はあるか？
- [ ] **削除表示**: 不要に思える情報はあるか？
- [ ] **レイアウト**: 画面レイアウトは見やすいか？
- [ ] **にぎやかさ**: 情報量は多すぎ/少なすぎ？

## 🎯 実装への活用

このプロトタイプをベースに、以下の方針で実装を進めます：

1. **フィードバック適用** → UI/UX を修正
2. **API 実装** → サンプルデータを API データに置き換え
3. **ロジック実装** → 計算ロジック、通知機能の実装
4. **テスト追加** → Jest & Playwright テスト
5. **本番デプロイ** → GitHub にコミット・プッシュ

## ❓ よくある質問

**Q: プロトタイプ画面で何もできないのですが？**
A: 設定は画面上で変更・保存できますが（見た目のみ）、実際にデータベースに保存されません。トースト通知が表示される動作のみ確認できます。

**Q: スマートフォンで見たい**
A: localhost:3000/prototype にアクセスしてください。レスポンシブ対応しています。

**Q: 画面のコードを修正したい**
A: 以下のファイルを編集してください：
- `src/components/ReminderSettingsPrototype.tsx`
- `src/components/MilestonesListPrototype.tsx`
- `src/app/prototype/page.tsx`

修正後は npm run dev を再実行してください。
